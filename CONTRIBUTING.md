# Contributing to PlyDB

First off, thank you for considering contributing! It’s people like you who make
PlyDB better for everyone.

To keep the project healthy and sustainable, we use an **Open Core** model. This
means the core of the engine is open-source under the **Apache 2.0 License**,
while leaving space for a separate commercial version with advanced features to
fund continued development.

---

## The Contributor License Agreement (CLA)

So that we can offer a commercial version of PlyDB, we require all contributors
to sign our **Contributor License Agreement (CLA)** before we can merge your
code.

By submitting a Pull Request, you agree to the terms of our
[Contributor License Agreement](/CLA.md)

By agreeing with the CLA, you grant us the right to include your contributions
in both the open-source version and our proprietary version. This ensures we
have the legal "clearance" to distribute the software without future licensing
conflicts.

---

## How to Contribute

### 1. Report a Bug

If you find a bug, please open an **Issue**. Be sure to include:

- A clear description of the problem.
- Steps to reproduce the issue.
- Your environment (OS, version, etc.).

### 2. Suggest a Feature

We love ideas! However, before getting too deep into implementation, please
first check our [Roadmap](/TODO.md) and start a discussion so we can align on a
direction.

Features to improve PlydB for individuals can and should be a part of this open
source version, while features aimed to improve usage at scale, as in an
enterprise, should be reserved for the proprietary version. Doing so helps keep
the project sustainable. If you aren't sure where a feature fits, feel free to
open a discussion first!

### 3. Submit a Pull Request

1. Fork the repository.
2. Create a new branch (`git checkout -b feature/cool-new-thing`).
3. Commit your changes.
4. Push to the branch and open a Pull Request.
