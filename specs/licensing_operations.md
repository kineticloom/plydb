
## File headers

All source files (`.go`, `.sh`, `.ps1`) include a 2-line SPDX header. The short
SPDX-only form is used to minimise boilerplate:

```
// Copyright 2026 Paul Tzen
// SPDX-License-Identifier: Apache-2.0
```

Shell files use `#` comments; PowerShell files use `<# ... #>` blocks.

## Build targets

### `make license-check`

Runs two checks:

1. **Header check** — `github.com/google/addlicense -check -s=only` verifies
   every source file has the correct SPDX header.
2. **Dependency allowlist** — `github.com/google/go-licenses check` verifies all
   transitive Go dependencies use an approved license. Allowed licenses:
   `Apache-2.0`, `MIT`, `BSD-2-Clause`, `BSD-3-Clause`, `ISC`, `MPL-2.0`.

### `make license-fix`

Adds or corrects SPDX headers in all source files using
`github.com/google/addlicense -s=only`. Run this locally after creating new
files or when `license-check` reports missing headers.

### `make notices-generate`

Generates `THIRD_PARTY_NOTICES.md` from `scripts/notices.tpl` using
`github.com/google/go-licenses report`. Run locally whenever dependencies change
(`go get`, `go mod tidy`).

### `make notices-check`

Regenerates notices to a temp file and diffs against the checked-in
`THIRD_PARTY_NOTICES.md`. Exits non-zero if the file is stale. Used in CI.

## Third-party notices

`THIRD_PARTY_NOTICES.md` documents the licenses of all transitive Go
dependencies. It is generated by `make notices-generate` using the Go template
at `scripts/notices.tpl`, which produces one Markdown section per dependency
with its name, version, SPDX identifier, license URL, and full license text.

The file must be kept up to date and is verified by `make notices-check` on
every pull request.

## Pull requests

`.github/workflows/pr.yml` runs a `Run checks` job on every PR targeting `main`,
which includes:

- Build
- Vet
- Unit tests
- Integration tests
- License check
- Notices check

## CLA

`.github/workflows/cla.yml` runs on `pull_request_target` and `issue_comment`
events using `contributor-assistant/github-action@v2.6.1`. Signatures are stored
in `signatures/version1/cla.json`. The CLA document is at `CLA.md`. Contributors
sign by commenting `I have read the CLA Document and I hereby sign the CLA`. The
allowlist exempts `ypt` and `dependabot[bot]`.
